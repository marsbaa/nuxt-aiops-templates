# Pre-code Checklist

- [ ] Problem understood: PRD/feature spec read and clarified.
- [ ] Scope agreed: goals/non-goals captured; assumptions listed.
- [ ] UX/UI ready: designs/wireframes reviewed; states defined (loading/empty/error).
- [ ] Contracts ready: OpenSpec API definitions reviewed/updated.
- [ ] Data model confirmed: Firestore collections, indexes, security rules impact assessed.
- [ ] Architecture drafted: components, stores, composables, services mapped.
- [ ] Dependencies identified: external APIs, libraries, flags, migrations.
- [ ] Testing plan set: acceptance criteria captured; test types defined.
- [ ] Observability plan: logs/metrics/traces/events defined.
- [ ] Rollout plan: feature flag strategy, environments, and gating agreed.
- [ ] Risks captured: with mitigations and owners.
- [ ] Definition of Done agreed for the task/feature.
